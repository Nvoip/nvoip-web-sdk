SDK de autenticação

Validação pronta para entrar no fluxo do cliente.

Use o mesmo componente para validar telefone por OTP ou exigir 2FA depois da senha. O frontend nunca recebe credenciais da API da Nvoip; ele conversa apenas com o backend da aplicação.

1Login aprovado no backend chama 2FA.
2OTP valida telefone antes de avançar no cadastro.
3Após validar, o SDK redireciona para a URL de retorno.

Login do cliente

Acesse o painel

Neste exemplo, a senha já foi aceita pelo backend. Ao continuar, o SDK abre o 2FA e só libera o acesso depois do código validado.

Uso no app real: chame NvoipAuthWidget.start2FA() depois do login primário e informe returnTo para concluir a sessão.
